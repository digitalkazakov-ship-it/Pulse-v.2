# Competify Radar — локальный дашборд

Конкурентная аналитика по категориям. React + Vite + TypeScript фронтенд, данные загружаются из JSON-файлов в `public/data/`, которые генерируются Python-скриптами из Excel-выгрузок.

---

## Запуск фронтенда

```bash
npm install
npm run dev
# → http://localhost:8080
```

---

## Обновление данных

Каждый раздел дашборда питается от отдельного скрипта. При получении нового Excel-файла:

1. Положить файл в нужную папку (пути прописаны внутри скрипта)
2. Запустить скрипт: `py scripts/<имя_скрипта>.py`
3. Скрипт перезаписывает JSON в `public/data/`
4. Обновить в скрипте константы `CURR`, `PREV`, `YOY` (периоды сравнения), если сменился отчётный месяц

Python-зависимость: `openpyxl`. Установка: `pip install openpyxl`.

## Скрипты и правила сбора данных

### 1. `scripts/process_digital.py` → `public/data/digital_audit.json`

**Раздел дашборда:** Site Stats

**Входные файлы:**

| Переменная  | Файл                  | Содержание             |
|-------------|-----------------------|------------------------|
| `FILE_CURR` | `report act.xlsx`     | Текущий год (янв–апр 2026) |
| `FILE_PREV` | `report act 2.xlsx`   | Предыдущий год (янв–дек 2025) |

**Вкладки Excel и метрики:**

| Вкладка                  | Агрегация | Метрика в JSON          |
|--------------------------|-----------|-------------------------|
| `Visits`                 | sum       | Monthly Visits (K)      |
| `Unique_Visits`          | sum       | Unique Visitors (K)     |
| `Pages_Per_Visit`        | mean      | Pages/Visit             |
| `Avg_Visit_Duration`     | mean      | Avg Duration (min)      |
| `Bounce_Rate`            | mean      | Bounce Rate (%)         |
| `Desktop_Overview_Share` | —         | Источники трафика (desktop) |
| `Mobile_Overview_Share`  | —         | Источники трафика (mobile)  |

**Бренды (строка `site` в файле → ключ):**

| Домен          | Ключ   | Название   |
|----------------|--------|------------|
| `theact.ru`    | BrandX | The Act    |
| `librederm.ru` | CompA  | Librederm  |
| `mixit.ru`     | CompB  | Mixit      |
| `vois.ru`      | CompC  | Vois       |

**Периоды сравнения** (менять при смене отчётного месяца):

```python
CURR, PREV, YOY = '2026-04', '2026-03', '2025-04'
T6  = ['2025-11', ..., '2026-04']   # спарклайны (6 мес.)
T12 = ['2025-05', ..., '2026-04']   # график посещений (12 мес.)
```

**Правила для источников трафика:**
- Берётся последний доступный месяц в файле
- `Organic Search` и `Paid Search` объединяются в канал `Search`
- `Paid Search` и `Display Ads` → paid; остальные → organic
- Мультибрендовые строки не используются (фильтр по `BRAND_MAP`)

### 3. `scripts/process_ad_spend.py` → `public/data/ad_spend.json`

**Раздел дашборда:** Media & Creatives → график «Рекламные расходы»

**Входной файл:** `flowchart_Mineral water_28.05.2026.xlsx`, вкладка `Flowchart`

**Фильтр по дате:** строки, где `Неделя` (дата начала недели) ≥ `2026-12-01`.
Текущий диапазон: декабрь 2025 — апрель 2026 (5 месяцев).

**Используемые колонки:**

| Колонка              | Описание                          |
|----------------------|-----------------------------------|
| Список брендов       | Определение бренда                |
| Тип медиа            | Определение канала                |
| Неделя               | Дата недели → `YYYY-MM` для агрегации |
| Prometheus Est. Cost | Стоимость размещения в рублях     |

**Агрегация:** сумма `Prometheus Est. Cost` по каждому бренду × месяц × канал, делённая на 1 000 000 (млн руб.).

**Маппинг каналов** — те же ключи, что и в `process_creatives.py`: `tv`, `radio`, `outdoor`, `digital`. В графике вкладка `outdoor` отображается как «Наружная реклама».

**Маппинг брендов и правило мультибрендовых строк** — аналогично `process_creatives.py`.

**Смена периода:** изменить константу `DATE_FROM = datetime.datetime(2025, 12, 1)`.

### 4. `scripts/process_sales.py` → `public/data/sales.json`

**Раздел дашборда:** Sales

**Входной файл:** `Питьевая вода 01.01.2021-30.04.2026.xlsx`, вкладка `Показатели`

**Структура файла:** строка 0 — названия метрик, строка 1 — названия брендов, строки 2+ — данные. Дата в колонке 0, формат `DD.MM.YYYY`.

**Фильтр по дате:** апрель 2025 — апрель 2026 (13 месяцев).

**Используемые колонки:**

| Колонки | Метрика                                    | График                |
|---------|--------------------------------------------|-----------------------|
| 1–4     | Средняя цена, ₽                            | Цена бренда           |
| 5–8     | Индекс продаж (уп.), k                     | Индекс продаж         |
| 9–12    | Доля бренда в категории (уп.), %           | Доля в категории      |
| 13–16   | Динамика продаж (шт.) год к году, %        | Динамика продаж       |
| 17–20   | Дистрибуция, %                             | Офлайн-дистрибуция    |

Порядок брендов в каждой группе колонок: Нарзан → Borjomi → Сенежская → Святой Источник.

**Определения метрик:**
- **Индекс продаж** — объём продаж бренда в упаковках / средний объём всей категории в 2020. Значения ~0.01–0.12.
- **Динамика продаж** — (продажи текущего периода / продажи аналогичного периода год назад) − 1, в %. Может быть отрицательной.

**Смена периода:** изменить `DATE_FROM` и `DATE_TO` в скрипте.

### 7. `scripts/process_perception.py` → `public/data/perception.json`

**Раздел дашборда:** Perception & Social Listening

**Входной файл:** `Social listening/SL.xlsx`

---

#### Вкладка `Шаг 4.1 - SL` — 3 графика SL + sentiment

Используются только **ПОЗИТИВ абс.** (cols D–I, строки 4–15) и **НЕГАТИВ абс.** (cols P–U, строки 4–15).
CRM & программа лояльности и Оффлайн магазины — **пропускаются** (нули).

**Категории и строки (0-based):**

| Категория | Параметр | Строка |
|-----------|----------|--------|
| Ассортимент | Уникальные и эксклюзивные продукты | 4 |
| | Широкий ассортимент | 5 |
| | Высокое качество товаров | 6 |
| | Высокое качество упаковки | 7 |
| Цена | Конкурентоспособные и выгодные цены | 8 |
| | Наличие скидок и промо | 9 |
| Продукт | Приятный вкус | 13 |
| | Польза от употребления | 14 |
| | Можно часто пить | 15 |

**Бренды — ПОЗИТИВ (col index 0-based):** Нарзан=3, Свят.ист=5, Сенежская=6, Боржоми=8.
**Бренды — НЕГАТИВ:** Нарзан=15, Свят.ист=17, Сенежская=18, Боржоми=20.

**Тотал по категории** = сумма sub-params (не строка 16 «Кол-во WOM»).

**Sentiment** = `(ΣПозитив − ΣНегатив) / (ΣПозитив + ΣНегатив) × 100` по всем трём категориям.
История накапливается: скрипт читает существующий `perception.json`, дописывает/обновляет текущий месяц и пересохраняет. Менять `GENERATED = 'YYYY-MM'` при каждом новом периоде.

---

#### Вкладка `Шаг 4.4 - Имидж` — 2 графика

**Таблица 1 (абсолютные значения)**, cols 0-based: Нарзан=2, Боржоми=3, Свят.ист=4, Сенежская=6. Строки 2–21 (20 характеристик).

**Таблица 3 (сравнение с медианой)**, cols 0-based: Нарзан=22, Свят.ист=24, Сенежская=25, Боржоми=27. Значения умножаются на 100 → %.

**Имиджевые характеристики** (7 метрик, строки): Надежный(2), Доступный(4), Дорогой(5), Современный(8), Натуральный(10), Вкусный(18), Лечебный(20).

**Матрица Бренд×Имидж**: все 20 характеристик, два режима — абсолют и % к медиане.

## Структура `public/data/`

```
public/data/
├── digital_audit.json   ← Site Stats (веб-аналитика)
├── creatives.json       ← Media & Creatives: мониторинг + сюжеты
├── ad_spend.json        ← Media & Creatives: рекламные расходы
├── sales.json           ← Sales: продажи, доля, цена, дистрибуция
├── wordstat.json        ← Brand Awareness: Яндекс Wordstat (апр 2024 – апр 2025)
├── neuro.json           ← Brand Awareness: выдача в нейроответах (Google + Yandex)
├── bht.json             ← Brand Awareness: BHT-метрики, воронка, возраст, частота (Q2 2025 – Q1 2026)
├── perception.json      ← Perception & Social: SL-графики, sentiment, имидж, матрица
├── ecom.json            ← Availability: продажи, выручка, SKU, доли (Ozon/WB/ЯМ)
└── presence.json        ← Availability: представленность в ритейлерах и сервисах доставки
```
